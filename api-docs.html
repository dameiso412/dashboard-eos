Overview

This API gives any AI system or external tool full bidirectional access to all data in the EOS Dashboard and CEO Dashboard. Built on Supabase PostgREST — every table is a REST endpoint. No backend required.

The platform stores two types of data:

  • EOS Dashboard — quarterly rocks, weekly scorecard, leaderboard, fame wall, revenue/utility pulso, team members
  • CEO Dashboard — monthly RAM with weekly breakdowns per metric (fully customizable sections & metrics)

Conventions

Query Filters

// Filter: eq, neq, gt, gte, lt, lte, like, in
?quarter_key=eq.2026-Q1
?year=eq.2026&month=eq.2       // March (0-indexed)
?id=in.(id1,id2,id3)

// Select specific columns
?select=id,name,status

// Order
?order=sort_order.asc

// Upsert (create or update by PK)
Prefer: resolution=merge-duplicates

Quarter Key Format

"2026-Q1"  // January–March 2026
"2026-Q2"  // April–June 2026
"2026-Q3"  // July–September 2026
"2026-Q4"  // October–December 2026

Month Index (RAM)

0=January  1=February  2=March  ... 11=December

AI Agent Pattern

Both Dashboards

This is the recommended pattern for an AI agent that monitors and updates the dashboard automatically:

// 1. READ current state
const rocks = await get('/rocks?quarter_key=eq.2026-Q1');
const offTrack = rocks.filter(r => r.status === 'off-track');

// 2. ANALYZE and decide action
if (offTrack.length > 1) {
    // 3. WRITE — update scorecard result
    await upsert('/scorecard_results', {
        id: `sr-rocks-on-track-${currentWeek}`,
        metric_id: 'rocks_on_track_metric_id',
        week: currentWeek,
        result: `${rocks.length - offTrack.length}/${rocks.length}`,
        good: offTrack.length === 0
    });

    // 4. WRITE — add a note to the RAM
    await upsert('/ram_data', {
        id: `clients_lost_2026_2`,
        metric_id: 'clients_lost',
        year: 2026, month: 2,
        note_week3_4: '2 rocks off track — priority review needed'
    });
}

💡 Tip for AI agents: Always use POST with Prefer: resolution=merge-duplicates (upsert) rather than PATCH, unless you are certain the row exists. Upsert is idempotent and safe to call multiple times.
